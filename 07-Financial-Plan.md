### B. API Integration Costs

#### Option 1: 1688 Official API (Recommended for Phase 2)

| API | Cost | Notes |
|-----|------|-------|
| **Product Search** (alibaba.offer.search) | **FREE** | QPS limited to 10-20/sec — sufficient for Phase 1 |
| **Product Detail** (alibaba.item.get) | **FREE** | |
| **Order APIs** (alibaba.trade.buyer.list) | **FREE** | Need permission application |
| **High-QPS & Real-time Inventory** | ¥980-1,980/year (~$135-$275/yr) | Optional resource pack for higher volume |

**Requirements:** 1688 Open Platform account + Chinese business verification (Touab's BD number or Chinese entity can handle this)

#### Option 2: Taobao Open Platform API

| API | Cost | Notes |
|-----|------|-------|
| Item search & detail | **FREE** (limited QPS) | Requires Taobao App Key |
| Higher rate limits | Negotiable | For commercial apps |

**Note:** Taobao API is more consumer-focused. For wholesale sourcing, **1688 API** is the right choice.

#### Option 3: Third-Party Aggregator (Alternative/Fallback)

| Provider | Pricing | Coverage | Best For |
|----------|---------|----------|----------|
| **OT Commerce** (otcommerce.com) | K0 Basic: ~$30-50/mo<br>K1 Standard: ~$100-200/mo<br>K2 Light: Pay-per-call | Taobao, 1688, Alibaba, Tmall, JD, AliExpress | Faster time-to-market, multi-platform data |
| **Onebound** (onebound.net) | ~¥5,000-50,000/yr depending on API call volume + source code licensing available | Taobao, 1688, Tmall | Full turnkey agent systems available (¥20K-200K) |
| **SimplyBatch / Accio** | Custom pricing | 1688 sourcing | BD-specific sourcing agents |

**Recommendation:** Start with **free 1688 official APIs** for Phase 1 (manual fallback for gaps). If that's too complex, use **OT Commerce K0 Basic ($30-50/mo)** as middle ground. Upgrade to paid packs only when order volume exceeds free limits.

#### Option 4: In-House Scraping (MVP Phase 1 Only)

| Tool | Cost | Notes |
|------|------|-------|
| Puppeteer/Playwright on server | Free (included in hosting) | More fragile, higher maintenance |
| Human fallback | Labor cost | For edge cases the scraper can't handle |

**For MVP, I recommend this hybrid approach:**
- Use free 1688 API for product search/detail where possible
- Use puppeteer scraping as backup for products not covered
- Manual human review for QC/correction
- Migrate to paid API packs when we hit Phase 2 scale

### E. Cost Comparison: Hosting Options

| Option | Monthly Cost (USD) | Monthly Cost (BDT) | Pros | Cons |
|--------|-------------------|-------------------|------|------|
| **Vercel + Supabase Pro** | ~$45-60 | ~5,500-7,200 | Zero DevOps, auto-scaling, preview deploys | Vendor lock-in, higher cost at scale |

**My Recommendation for Monir:**

- **Phase 1.5 (Post-launch):** Upgrade to Vercel Pro ($20) + Supabase Pro ($25) = ~$45/mo when we have paying customers
- **Alternative:** Your Oracle VM is a great option to save costs if you want to self-host. I'd still recommend Vercel for the frontend for easy deployment.
